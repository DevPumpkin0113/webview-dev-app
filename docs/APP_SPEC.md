# Flutter 기반 모바일 WebView 개발용 앱 기획서

## 1. 목적

본 앱은 **웹게임 및 웹콘텐츠 개발 중 모바일 환경에서의 실사용 감각을 빠르게 확인하기 위한 개발자 전용 앱**이다.

APK/IPA 포팅이나 PWA 설치 없이, **로컬 개발 서버 URL을 직접 입력하여 전체화면(WebView)으로 즉시 확인**하는 것을 목표로 한다.

본 앱은 **출시용 앱이 아닌 내부 개발/테스트 도구**이며, 사용자는 개발자 본인 또는 개발팀으로 한정한다.

---

## 2. 타겟 사용자

- 모바일 웹게임 개발자
- 웹 기반 인터랙티브 콘텐츠 개발자
- Phaser / Pixi / Three.js 등 웹 엔진 사용 개발자
- 모바일 UI/UX 감각을 빠르게 확인하고 싶은 개발자

---

## 3. 핵심 사용 시나리오

1. 개발자가 PC에서 로컬 개발 서버 실행
   - 예: `http://192.168.0.10:5173`

2. 모바일 기기에서 본 앱 실행
3. 상단 URL 입력창에 개발 서버 주소 입력
4. WebView로 즉시 로드
5. 전체화면 상태에서 게임/콘텐츠 테스트
6. URL 변경 또는 새로고침을 통해 다른 씬/페이지 테스트

---

## 4. 핵심 요구사항 (Functional Requirements)

### 4.1 WebView 기능

- Flutter `WebView` 사용
- HTTP / HTTPS URL 모두 허용
- JavaScript 실행 허용
- 로컬 네트워크(IP 기반) URL 접근 가능

### 4.2 URL 입력 UI

- 화면 상단에 개발자용 URL 입력 바 제공
- 입력 후 엔터 또는 로드 버튼으로 즉시 이동
- 최근 사용 URL 히스토리 최대 N개 저장 (옵션)

### 4.3 전체화면 모드

- 기본 진입 시 전체화면
- 상태바/시스템 UI 최소화
- 전체화면 ON/OFF 토글 가능

### 4.4 리로드 및 제어

- 새로고침 버튼 제공
- WebView 강제 reload 기능

---

## 5. 부가 기능 (Nice to Have)

- 최근 URL 목록 드롭다운
- 가로/세로 화면 고정 토글
- Safe Area 가이드 오버레이 표시
- FPS / 화면 해상도 표시 (디버그용)
- 콘솔 로그 미러링 (JS → Flutter)

---

## 6. 비기능 요구사항 (Non-Functional Requirements)

- **속도 우선**: 앱 실행 → URL 로드까지 최소 단계
- **안정성**: 크래시 없이 반복 로드 가능
- **단순성**: UI는 개발자 도구 수준으로 최소화
- **확장성**: 추후 WebView 옵션 추가 가능 구조

---

## 7. 기술 스택

- Framework: Flutter
- WebView: `webview_flutter`
- 상태관리: 필요 시 Provider / Riverpod (초기에는 불필요)
- 플랫폼: Android 우선, iOS 확장 가능

---

## 8. 화면 구성 (와이어프레임 개념)

```
┌────────────────────────────┐
│ URL 입력창  [ Load ] [ ↻ ] │  ← Dev Toolbar
├────────────────────────────┤
│                            │
│        WebView 영역         │
│                            │
│   (웹게임 / 웹콘텐츠)       │
│                            │
└────────────────────────────┘
```

- Dev Toolbar는 토글 가능
- 기본은 반투명 또는 자동 숨김

---

## 9. 플랫폼별 고려사항

### Android

- cleartextTraffic 허용 (HTTP 테스트용)
- 전체화면 immersive mode 지원

### iOS

- ATS 설정 필요 (로컬 HTTP 접근 시)
- Safe Area 처리 필수

## 10. 범위 외 (Out of Scope)

- 앱스토어 배포
- 사용자 계정 관리
- 푸시 알림
- 보안 인증

---

## 11. 성공 기준

- 로컬 개발 서버 URL을 모바일에서 즉시 로드 가능
- 주소창 없는 전체화면 상태로 테스트 가능
- URL 변경 → 재로드 흐름이 빠르고 스트레스 없음
